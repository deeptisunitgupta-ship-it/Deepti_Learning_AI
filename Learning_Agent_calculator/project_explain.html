1. Project Overview

This project teaches you how to build an AI Agent using LangGraph that can answer natural language math questions. Instead of computing the answer itself, the LLM (Google Gemini 2.5 Flash) reasons about which calculator tool to use and calls that tool — then explains the result in plain English.

What makes this an “Agent”?

A regular LLM just generates text. An Agent can take actions — it decides to call external tools, reads back the results, and uses those results to form its answer. This project is the simplest possible example of that pattern.

LangGraph Graph

create_react_agent() builds a compiled graph with two nodes: an agent node (the LLM) and a tools node (executes Python functions). Conditional edges decide whether to loop back (more tool calls needed) or end (final answer ready).

State = Message List

The shared state is simply a growing list of messages. Each step appends a new message. The LLM always reads the entire history to decide its next action. No memory needed — the conversation is the state.

LangGraph Internal Graph

     START

       v

    │                             │
    │ tool_call detected          │ no tool_call → final answer

  └──────┬───────┘
         │  result appended as ToolMessage
         └──────► back to agent node (LLM reads result)

3. How the ReAct Agent Works

ReAct = Reasoning + Acting. The LLM alternates between thinking about what to do and actually doing it.

1
REASON LLM reads the question and all previous messages. Decides which tool to call.
2
ACT LLM emits a tool_call: multiply(a=13, b=7). The tools node runs the Python function.
3
OBSERVE Tool returns 91.0. This is added to state as a ToolMessage.
4
RESPOND LLM reads the result and writes the final natural language answer. No more tools needed → END.
Key insight: The LLM never computes the math. It only decides which tool to call and what arguments to pass. All actual arithmetic is done by plain Python functions.

5. Calculator Tools — tools.py

Each tool is a plain Python function decorated with @tool from LangChain. The decorator registers the function as a callable tool — the LLM reads the docstring to understand what the tool does and when to use it.

add
(a: float, b: float) → float
Returns a + b
"What is 5 plus 3?"
subtract
(a: float, b: float) → float
Returns a - b
"Subtract 48 from 200"
multiply
(a: float, b: float) → float
Returns a * b
"13 multiplied by 7"
divide
(a: float, b: float) → float
Returns a / b. Raises error if b=0
"Divide 144 by 12"

How @tool works — annotated code

from langchain_core.tools import tool

@tool                              # ① registers this function as an LLM-callable tool
def multiply(a: float, b: float) -> float:
    """
    Multiply two numbers together and return the result.
    Use this tool when the user wants to multiply, find the product,
    or scale a number.
    """                            # ② LLM reads THIS docstring to decide when to call it
    return a * b                   # ③ plain Python — the LLM never sees this line


# Collect all tools into a list — passed to the agent
CALCULATOR_TOOLS = [add, subtract, multiply, divide]
Important: The LLM never sees the function body. It only sees: (1) the tool name, (2) the docstring, and (3) the parameter names and types. Write clear, descriptive docstrings — they directly affect how well the LLM uses your tools.

6. Agent Setup — agent.py

This file has one job: create and return the compiled LangGraph agent. It connects the Gemini LLM to the calculator tools and sets a system prompt. 

from langchain_google_genai import ChatGoogleGenerativeAI
from langgraph.prebuilt import create_react_agent
from tools import CALCULATOR_TOOLS

def create_calculator_agent():

    # Step 1 — create the Gemini 2.5 Flash LLM
    llm = ChatGoogleGenerativeAI(
        model="gemini-2.5-flash",      # model name
        google_api_key=api_key,        # loaded from .env
        temperature=0,               # 0 = deterministic (best for math)
    )

    # Step 2 — build the LangGraph ReAct agent
    agent = create_react_agent(
        model=llm,                     # the LLM that reasons
        tools=CALCULATOR_TOOLS,        # tools it can call
        prompt="You are a helpful calculator assistant..."  # system prompt
    )

    return agent  # a compiled LangGraph CompiledGraph object

Gemini 2.5 Flash

  • Google's fast, capable model
  • Excellent at tool calling
  • temperature=0 makes answers consistent
  • Free tier available via Google AI Studio

create_react_agent()

  • LangGraph prebuilt helper
  • Builds the full graph automatically
  • Handles the tool-calling loop internally
  • Returns a compiled graph ready to invoke

7. Entry Point — main.py

This file provides two modes and prints the complete agent thought trace.

Demo Mode (default)

Runs 5 pre-set questions automatically. Best for your first run — no typing needed.

python main.py

Chat Mode

Interactive loop — type any math question. Type quit to exit.

python main.py chat

Key function: run_query()

def run_query(agent, question: str):
    result = agent.invoke(
        {"messages": [HumanMessage(content=question)]}
        # ↑  wraps the question as the first message in state
    )
    # result["messages"] = complete list of all messages (human + AI + tool)
    print_step_trace(result["messages"])        # prints the full trace
    final = result["messages"][-1].content   # last message = final answer
    print(final)

8. Message Flow & State

LangGraph's state is a list of messages. Every step appends a new message. Here is the complete state for the question "What is 13 multiplied by 7?":

YOU What is 13 multiplied by 7?
LLM Decides to call tool → multiply(a=13, b=7)
TOOL Python executes 13 * 7 → returns 91.0
FINAL 13 multiplied by 7 is 91.

The print_step_trace() function in main.py loops through all these messages and prints each one, so you can see exactly how the agent reasoned at every step.

10. How to Run

Step 1 — Navigate to the project folder

cd "Learning_Agent_calculator"

Step 2 — Install dependencies

pip install -r requirements.txt

Go to aistudio.google.com/app/apikey, sign in with your Google account, and click Create API key.

Step 4 — Create your .env file

Create a file named .env in the project folder:

GOOGLE_API_KEY=paste_your_key_here

python main.py chat
